# Project TWIN Vision

最終更新: 2026-06-28

---

## Project TWIN とは

**「夜業界の成功を再現可能にする運営OS」** を目指すプロジェクトである。

チャットボットを作ることが目的ではない。

AI接客・AI採用・AIマッチング・AI分析を通して、
店舗運営をデータで改善し続ける仕組みを作ることが目的である。

---

## なぜ作るのか

夜業界には

- 優秀な店長
- 売れるキャスト
- 常連客を作る接客

など、経験でしか語られないノウハウが数多く存在する。

しかし、その多くは属人的であり、人が辞めると一緒に失われてしまう。

Project TWIN は、それらをデータとして蓄積し、誰でも再現できる形へ変えることを目指す。

「優れた判断」を保存するのではない。
**「判断の過程」と「その結果」を保存する。**

判断だけ残しても、未来の AI には理由が分からない。
応募者の会話・AI の採用カルテ・店長が確認したこと・店長メモ・採用後の結果——
この全プロセスが揃ったとき、はじめて「なぜその判断が当たったのか」を学習できる。

> 経験は、人が持つ限り失われる。
> 記録され、学習され、再現できるようになったとき、初めて知識になる。
> Project TWIN は、その知識を未来へ受け継ぐための基盤を作る。

---

## Mission

> 人の経験を、誰でも再現できる知識へ。

---

## Vision

> 夜業界で働く人が、経験だけに頼らず、データを活用して成功できる世界を作る。

## Project TWIN の 5 つの Phase

### Phase 1 — AI 接客 ✅

**来店前の不安を解消する。**

初めてのお客様は来店前に多くの不安を持つ。
料金・雰囲気・キャスト・予約方法——その一つひとつに丁寧に答え、
「行ってみたい」という気持ちを来店という行動に変える。

---

### Phase 2 — AI 採用 🚧

**応募者の可能性を可視化する。**

採用の勘は重要だが、見落としも多い。
問診データとスコアで「気づきにくい可能性」を引き出し、
店長の採用判断を補助する。
採用後の実結果を記録し、AI評価の精度を継続的に高める。

---

### Phase 3 — AI マッチング ⬜

**お客様とキャストの相性を最大化する。**

来店前の会話から好みや雰囲気を読み取り、
相性の良いキャストを推薦する。
初回来店の体験品質を上げ、リピート率を高める。

---

### Phase 4 — AI 分析 ⬜

**来店・売上・採用・定着を分析する。**

蓄積されたデータを横断分析し、
「なぜ売れたか」「なぜ定着したか」「なぜ離脱したか」を明らかにする。
成功パターンを数字で示し、経験則を検証可能な仮説へ変える。

---

### Phase 5 — 夜業界運営 OS ⬜

**店舗運営を継続的に改善する AI プラットフォームへ。**

接客・採用・マッチング・分析が一つのシステムに統合され、
店舗全体が自律的に改善サイクルを回せる状態を目指す。
複数店舗への横展開も視野に入れる。

## Project TWIN が扱うデータ

| データ | 取得元 | 用途 |
|---|---|---|
| 来店前会話 | チャットログ | intent 分析・FAQ改善・来店率向上 |
| 応募者問診 | 採用チャット | スコア算出・Grade判定 |
| 採用結果 | 店長入力 | AI評価精度の検証 |
| 売上ランク | 店長入力 | Grade vs 実力の相関分析 |
| 定着状況 | 店長入力 | 採用予測モデルの改善 |
| 呼客実績 | 店長入力 | 問診スコアの妥当性確認 |
| キャスト出勤情報 | WBSS API | AI接客の回答精度 |
| リピート・来店傾向 | 将来連携 | マッチング精度の向上 |

## AI は判断者ではない

Project TWIN は「人を評価する AI」ではない。**「店長の判断を支援する AI」** である。

最終判断は必ず人間が行う。
AI は判断材料を整理し、より良い意思決定を支援する。

採用スコアは「この人を採るべきか」を決めるものではなく、
「この人のどこを面接で確認すべきか」を示すものである。

## WBSS・LENS・Project TWIN

```
WBSS（店舗運営・送迎・出勤管理）
        ↓ 出勤情報・キャストデータ
Project TWIN（AI接客・AI採用・AI分析）
        ↓ 接客データ・採用実績・改善提案
LENS（集客・SEO・MEO分析）
        ↓ 集客経路・来店転換率
        ↓
     運営改善
```

3 つのシステムが連携することで、「集客 → 来店 → 採用 → 定着 → 売上」という店舗の全サイクルを最適化する。

## 10 年後の目標

**Project TWIN が夜業界の標準プラットフォームになること。**

経験がデータになり、
成功パターンが蓄積され、
新人でも優秀な店舗運営ができる世界を実現する。

業界の知識を特定の店長だけが持つ時代を終わらせ、
誰もが学べる形で未来へ受け継ぐ。

## 開発方針

新機能を追加する前に、以下の順で判断する。

1. **収益性** — 店舗の売上・採用効率・運営コストに貢献するか
2. **実装可能性** — 小さなチームで現実的に作れるか
3. **運用コスト** — 完成後も維持できるか。障害時に素早く対応できるか
4. **継続性** — 1 年後も使われ続けるか。データが蓄積されるほど価値が上がるか
5. **差別化** — 他社が短期間で真似できないか

派手な機能ではなく、**現場で使われ続ける機能** を作る。

## Project TWIN が本当に残したいもの

Project TWIN は「AI プロジェクト」ではない。**「知識保存プロジェクト」** である。

AI はあくまで手段だ。本当に残したいのは、

- 店長の判断とその根拠
- キャストの成功体験
- お客様との相性と接客の工夫
- 店舗運営の現場の知恵

そういう、人が持っている限り失われ続けるものである。

夜業界だけではない。ベテラン営業・熟練職人・名料理人・優秀な看護師——
あらゆる現場で「感覚で分かる」ノウハウは、本人が辞めると消えていく。
それをロストテクノロジーにしないことが、このプロジェクトの使命のひとつである。

AI が進化すれば、保存された知恵をより深く読み解けるようになる。
しかし、知恵そのものが保存されていなければ、未来の AI でも学べない。
だから今、情報を失わずに残すことを優先する。

## 最後に

Project TWIN は、AI を作るプロジェクトではない。

> **現場で生まれる知恵と判断を、100年後の AI でも学べる形で残すためのプロジェクトである。**

「100年」は象徴的な数字だ。伝えたいのは、「今の AI だけを相手にして設計しない」という姿勢である。
未来のモデルが今よりはるかに賢くなったときでも活かせるように、まず情報を失わずに残す。
解釈は、未来のより賢い AI に任せる。

---

## 100年後の AI でも学べる形で残す

Project TWIN は、判断結果だけを保存しない。

応募者との会話・AI が生成した採用カルテ・店長が実際に確認した項目・店長メモ・採用後の結果——
これらを一連の流れとして保存する。

なぜなら、未来の AI が学習できるのは「結論」ではなく、
**「判断に至る過程」と「その結果」** だからである。

```
判断だけ保存する       → 短期的には楽。でも将来のAIが理由を学べない
会話だけ保存する       → 文脈は残る。でも結果と結びつかない
店長メモだけ保存する   → 人間の感覚は残る。でも再現性が弱い
全プロセスを保存する   → 判断プロセス全体が教師データになる
```

今の AI で完全に解釈できなくてもよい。
大切なのは、未来の AI が再解釈できるように、
**現場の言葉と判断の文脈を失わずに残すこと** である。

これが Project TWIN のデータ設計哲学である。

> 事実を先に保存し、解釈は後から行う。
